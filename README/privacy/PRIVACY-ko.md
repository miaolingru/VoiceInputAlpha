# AtomVoice 개인정보 처리방침

최종 업데이트: 2026년 5월 5일

AtomVoice는 macOS 메뉴 바 음성 입력 도구입니다. 우리는 귀하의 개인정보를 중요하게 생각합니다. 본 개인정보 처리방침은 AtomVoice가 데이터를 처리하고, 권한을 사용하며, 타사 서비스와 상호작용하는 방법을 설명합니다.

## 1. 핵심 원칙

AtomVoice는 기기에서 로컬로 데이터를 처리하고 데이터 수집을 최소화하도록 설계되었습니다.

AtomVoice는 사용자 계정을 운영하지 않고, 광고를 표시하지 않으며, 분석 SDK를 포함하지 않고, 사용자 행동을 추적하지 않으며, 개인정보를 판매, 대여 또는 공유하지 않습니다.

## 2. 처리하는 데이터

AtomVoice는 작동 중 다음과 같은 데이터를 처리할 수 있습니다:

1. **음성 오디오**
   트리거 키를 눌러 녹음을 시작하면 AtomVoice가 마이크에 접근하고 음성 인식 및 파형 표시를 위해 음성 오디오를 처리합니다. 녹음이 끝나면 AtomVoice는 오디오를 로컬 파일로 저장하거나 AtomVoice 서버에 업로드하지 않습니다.

2. **인식 텍스트**
   음성 인식 결과는 일시적으로 플로팅 캡슐 창에 표시되고 녹음이 끝난 후 현재 입력 필드에 주입됩니다. AtomVoice는 인식 텍스트 기록을 저장하지 않습니다.

3. **클립보드 내용**
   인식 텍스트를 현재 커서 위치에 입력하기 위해 AtomVoice는 시스템 클립보드를 일시적으로 사용하여 붙여넣기 작업을 수행합니다. 앱은 주입 전에 원래 클립보드 내용을 일시적으로 저장하고 주입 후 원래 내용을 복원합니다. 클립보드 내용은 로컬 메모리에서만 짧은 시간 동안 유지되며 AtomVoice 서버에 업로드되지 않습니다.

4. **접근성 관련 정보**
   AtomVoice는 macOS 접근성 권한을 사용하여 트리거 키를 감지하고, 현재 입력 위치를 확인하며, 붙여넣기 작업을 시뮬레이션합니다. 앱은 키 입력을 기록하거나 다른 애플리케이션의 텍스트를 지속적으로 읽지 않습니다. 중복 구두점 회피와 같은 입력 경험 최적화를 위해 필요한 경우에만 현재 입력 필드의 커서 근처 정보를 읽습니다.

5. **로컬 설정**
   AtomVoice는 언어, 인식 엔진, 트리거 키, 입력 장치, 애니메이션 스타일, 무음 자동 중지 설정, LLM 공급자 URL, 모델 이름, 사용자 정의 프롬프트 등의 앱 설정을 로컬에 저장합니다. 이러한 설정은 macOS 로컬 환경설정에 저장됩니다.

6. **LLM API Key**
   LLM 텍스트 최적화를 활성화하고 API Key를 입력하면 AtomVoice는 해당 API Key를 로컬 설정에 저장하고 선택한 LLM 공급자에 대한 요청에만 사용합니다. AtomVoice는 API Key를 AtomVoice 서버에 업로드하지 않습니다.

## 3. 음성 인식 작동 방식

AtomVoice는 다양한 인식 모드를 지원합니다:

1. **Apple 음성 인식**
   기본적으로 AtomVoice는 Apple Speech 프레임워크를 사용하여 음성 인식을 수행합니다. macOS 버전, 언어, 시스템 기능에 따라 음성 인식은 기기에서 수행되거나 Apple의 음성 인식 서비스에 의해 처리될 수 있습니다. 관련 데이터 처리는 Apple의 개인정보 처리방침의 적용을 받습니다.

2. **Apple 기기 내 인식 모드**
   "Apple 기기 내 인식"을 활성화하고 현재 언어가 지원되는 경우 AtomVoice는 시스템에 기기 측 인식을 요청합니다.

3. **Sherpa ONNX 로컬 인식**
   로컬 Sherpa ONNX 인식 모델을 구성하면 음성 인식이 기기에서 완전히 수행되며 클라우드 인식 서비스에 업로드할 필요가 없습니다.

## 4. LLM 텍스트 최적화

LLM 텍스트 최적화는 기본적으로 비활성화되어 있습니다.

이 기능을 활성화하면 AtomVoice는 현재 인식된 텍스트를 구성된 LLM 공급자에 보내 오류 수정, 구두점 보완 또는 음성 전사 개선을 수행합니다. 지원되는 공급자에는 OpenAI, Anthropic, DeepSeek, Moonshot, Alibaba Cloud Bailian, Zhipu AI, Lingyi Wanwu, Groq, 사용자 정의 OpenAI-compatible API 또는 로컬 Ollama가 포함됩니다.

LLM 공급자로 전송되는 데이터에는 일반적으로 다음이 포함됩니다:

1. 현재 세션의 인식 텍스트
2. 시스템 프롬프트 또는 사용자 정의 프롬프트
3. 구성한 모델 이름
4. 인증을 위한 API Key

이 데이터의 처리 방식은 선택한 LLM 공급자에 따라 다릅니다. 사용 전 해당 공급자의 개인정보 처리방침 및 데이터 이용 약관을 확인하시기 바랍니다.

LLM 텍스트 최적화를 활성화하지 않으면 AtomVoice는 인식 텍스트를 LLM 공급자에게 전송하지 않습니다.

## 5. 자동 업데이트 확인

AtomVoice는 GitHub Releases를 통해 새 버전을 확인합니다. 업데이트 확인 시 앱은 최신 버전 정보를 GitHub에 요청합니다. GitHub는 자체 정책에 따라 IP 주소, 기기 네트워크 정보, User-Agent 등의 네트워크 요청 정보를 수신할 수 있습니다.

AtomVoice는 업데이트 확인 시 녹음, 인식 텍스트, 클립보드 내용 또는 LLM API Key를 전송하지 않습니다.

## 6. 필요한 권한

AtomVoice는 다음과 같은 macOS 권한이 필요합니다:

1. **마이크 권한**
   음성 인식을 위해 음성을 녹음하는 데 사용됩니다.

2. **음성 인식 권한**
   Apple Speech 프레임워크를 호출하여 음성을 텍스트로 변환하는 데 사용됩니다.

3. **접근성 권한**
   트리거 키 감지, 입력 위치 확인 및 인식 텍스트를 현재 애플리케이션에 주입하는 데 사용됩니다.

macOS 시스템 설정에서 언제든지 이러한 권한을 취소할 수 있습니다. 권한을 취소하면 관련 기능이 제대로 작동하지 않을 수 있습니다.

## 7. 데이터 저장 및 삭제

AtomVoice는 음성 녹음을 저장하지 않고, 음성 인식 기록을 저장하지 않으며, 사용자 계정을 생성하지 않습니다.

로컬에 저장되는 데이터는 주로 앱 설정입니다. 다음 방법으로 관련 데이터를 삭제할 수 있습니다:

1. 앱 내에서 LLM 설정을 지우거나 수정
2. macOS에서 AtomVoice 앱 환경설정 삭제
3. 앱과 관련 로컬 지원 파일 삭제

타사 LLM 서비스나 Apple 음성 인식을 사용하는 경우 해당 공급자의 정책에 따라 관련 데이터를 관리하거나 삭제하시기 바랍니다.

## 8. 데이터 공유

AtomVoice는 개인정보를 판매, 대여 또는 거래하지 않습니다.

데이터가 타사에 전송되는 경우는 다음과 같습니다:

1. Apple 음성 인식을 사용할 때 오디오 또는 인식 요청이 Apple에 의해 처리될 수 있습니다
2. LLM 텍스트 최적화가 활성화되면 인식 텍스트가 선택한 LLM 공급자에게 전송됩니다
3. 업데이트 확인 시 앱이 GitHub Releases에 액세스합니다
4. 사용자 정의 API 엔드포인트를 사용할 때 데이터가 구성한 서버로 전송됩니다

## 9. 보안 조치

AtomVoice는 데이터 처리를 최소화하고 기기 내 처리를 우선시합니다. 온라인 요청은 일반적으로 HTTPS를 통해 전송되지만, 로컬 Ollama 또는 기타 HTTP 주소와 같은 사용자 정의 API 엔드포인트를 구성한 경우 해당 서비스의 보안을 직접 확인하시기 바랍니다.

LLM API Key를 적절히 관리하고 신뢰할 수 없는 기기나 공유 계정 환경에 인증 정보를 저장하지 마시기 바랍니다.

## 10. 아동의 개인정보

AtomVoice는 일반 macOS 사용자를 대상으로 하며 아동을 특별히 대상으로 하지 않습니다. 아동의 개인정보를 의도적으로 수집하지 않습니다.

## 11. 방침 변경

앱 기능 변경에 따라 본 개인정보 처리방침을 업데이트할 수 있습니다. 주요 변경 사항은 프로젝트 페이지, 릴리스 노트 또는 앱 내 공지를 통해 안내합니다.
