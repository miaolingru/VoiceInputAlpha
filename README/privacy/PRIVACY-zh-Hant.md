# AtomVoice 隱私政策

最後更新：2026 年 5 月 5 日

AtomVoice 是一款 macOS 選單列語音輸入工具。我們非常重視你的隱私。本隱私政策說明 AtomVoice 如何處理資料、使用權限以及與第三方服務互動。

## 1. 核心原則

AtomVoice 的設計原則是盡量在本機完成處理，並盡量減少資料收集。

AtomVoice 自身不營運帳號系統，不提供廣告，不內建分析 SDK，不追蹤使用者行為，不出售、出租或共享使用者個人資訊。

## 2. 我們處理哪些資料

AtomVoice 在運作過程中可能處理以下資料：

1. **語音音訊**
   當你按住觸發鍵開始錄音時，AtomVoice 會存取麥克風並處理當前錄音音訊，用於語音辨識和音量波形顯示。錄音結束後，音訊不會被 AtomVoice 儲存到本機檔案，也不會上傳到 AtomVoice 自有伺服器。

2. **辨識文字**
   語音辨識結果會暫時顯示在浮動膠囊視窗中，並在錄音結束後注入到當前輸入框。AtomVoice 不會儲存你的辨識文字歷史紀錄。

3. **剪貼簿內容**
   為了將辨識文字輸入到當前游標位置，AtomVoice 會暫時使用系統剪貼簿執行貼上操作。應用程式會在注入前暫時儲存原剪貼簿內容，注入後盡量還原原內容。剪貼簿內容僅在本機記憶體中短暫處理，不會上傳到 AtomVoice 伺服器。

4. **輔助使用相關資訊**
   AtomVoice 使用 macOS 輔助使用權限監聽觸發鍵、偵測當前輸入位置，並模擬貼上操作。應用程式不會記錄你的按鍵內容，也不會持續讀取其他應用程式中的文字內容。僅在必要時讀取當前聚焦輸入框的游標附近資訊，用於避免重複標點符號等輸入體驗最佳化。

5. **本機設定**
   AtomVoice 會在本機儲存應用程式設定，例如語言、辨識引擎、觸發鍵、輸入裝置、動畫樣式、靜音停止設定、LLM 服務商位址、模型名稱、自訂提示詞等。這些設定儲存在 macOS 本機偏好設定中。

6. **LLM API Key**
   如果你主動開啟 LLM 文字最佳化並填入 API Key，AtomVoice 會將該 API Key 儲存在本機設定中，並僅用於向你選擇的 LLM 服務商發起請求。AtomVoice 不會將 API Key 上傳到 AtomVoice 自有伺服器。

## 3. 語音辨識如何處理

AtomVoice 支援不同的辨識方式：

1. **Apple 語音辨識**
   預設情況下，AtomVoice 使用 Apple Speech 框架進行語音辨識。根據 macOS、語言和系統能力，語音辨識可能在裝置端完成，也可能由 Apple 的語音辨識服務處理。相關資料處理受 Apple 隱私政策約束。

2. **Apple 本機辨識模式**
   如果你開啟「Apple 本機辨識」且當前語言支援，AtomVoice 會要求系統使用裝置端辨識。

3. **Sherpa ONNX 本機辨識**
   如果你選擇並設定本機 Sherpa ONNX 辨識模型，音訊辨識會在本機完成，不需要上傳到雲端辨識服務。

## 4. LLM 文字最佳化

LLM 文字最佳化預設關閉。

如果你主動開啟該功能，AtomVoice 會將本次辨識出的文字傳送給你設定的 LLM 服務商，用於糾錯、補全標點符號或最佳化語音轉寫錯誤。可能使用的服務商包括 OpenAI、Anthropic、DeepSeek、Moonshot、阿里雲百煉、智譜 AI、零一萬物、Groq、自訂 OpenAI-compatible API 或本機 Ollama。

傳送給 LLM 服務商的資料通常包括：

1. 本次辨識文字
2. 系統提示詞或自訂提示詞
3. 你設定的模型名稱
4. 認證所需的 API Key

這些資料的處理方式由你選擇的 LLM 服務商決定。請在使用前閱讀對應服務商的隱私政策和資料使用條款。

如果你不啟用 LLM 文字最佳化，AtomVoice 不會將辨識文字傳送給這些 LLM 服務商。

## 5. 自動更新檢查

AtomVoice 會透過 GitHub Releases 檢查是否有新版本。檢查更新時，應用程式會向 GitHub 請求最新版本資訊。GitHub 可能會根據其自身政策接收網路請求資訊，例如 IP 位址、裝置網路資訊和 User-Agent。

AtomVoice 不會在更新檢查中主動傳送你的錄音、辨識文字、剪貼簿內容或 LLM API Key。

## 6. 權限說明

AtomVoice 需要以下 macOS 權限：

1. **麥克風權限**
   用於錄製你的語音並進行語音辨識。

2. **語音辨識權限**
   用於呼叫 Apple Speech 框架將語音轉換為文字。

3. **輔助使用權限**
   用於監聽觸發鍵、偵測輸入位置，並將辨識文字注入當前應用程式。

你可以隨時在 macOS「系統設定」中撤銷這些權限。撤銷後，相關功能可能無法正常使用。

## 7. 資料儲存與刪除

AtomVoice 不儲存錄音檔案，不儲存語音辨識歷史紀錄，也不建立使用者帳號。

本機儲存的資料主要是應用程式設定。你可以透過以下方式刪除相關資料：

1. 在應用程式內清空或修改 LLM 設定
2. 刪除 macOS 中 AtomVoice 的應用程式偏好設定
3. 刪除應用程式及其相關本機支援檔案

如果你使用第三方 LLM 服務或 Apple 語音辨識服務，請根據對應服務商的政策管理或刪除相關資料。

## 8. 資料共享

AtomVoice 不會出售、出租或交易你的個人資料。

只有在以下情況下，資料可能被傳送至第三方：

1. 使用 Apple 語音辨識時，音訊或辨識請求可能由 Apple 處理
2. 主動開啟 LLM 文字最佳化時，辨識文字會傳送給你選擇的 LLM 服務商
3. 檢查更新時，應用程式會存取 GitHub Releases
4. 使用自訂 API 位址時，資料會傳送至你自行設定的服務端

## 9. 安全措施

AtomVoice 盡量減少資料處理範圍，並優先在本機完成任務。線上請求通常透過 HTTPS 傳送，但如果你設定了自訂 API 位址，例如本機 Ollama 或其他 HTTP 位址，請自行確認該服務的安全性。

請妥善保管你的 LLM API Key，避免在不可信裝置或共用帳號環境中儲存敏感憑證。

## 10. 兒童隱私

AtomVoice 面向一般 macOS 使用者，不專門面向兒童。我們不會有意收集兒童的個人資訊。

## 11. 隱私政策變更

我們可能會根據應用程式功能變更更新本隱私政策。重大變更會透過專案頁面、發布說明或應用程式內說明進行提示。
