# AtomVoice 隐私政策

最后更新：2026 年 5 月 5 日

AtomVoice（中文名：原子微语）是一款 macOS 菜单栏语音输入工具。我们非常重视你的隐私。本隐私政策说明 AtomVoice 如何处理数据、使用权限以及与第三方服务交互。

## 1. 核心原则

AtomVoice 的设计原则是尽量在本机完成处理，并尽量减少数据收集。

AtomVoice 自身不运营账号系统，不提供广告，不内置分析 SDK，不追踪用户行为，不出售、出租或共享用户个人信息。

## 2. 我们处理哪些数据

AtomVoice 在运行过程中可能处理以下数据：

1. **语音音频**
   当你按住触发键开始录音时，AtomVoice 会访问麦克风并处理当前录音音频，用于语音识别和音量波形显示。录音结束后，音频不会被 AtomVoice 保存到本地文件，也不会上传到 AtomVoice 自有服务器。

2. **识别文本**
   语音识别结果会临时显示在浮动胶囊窗口中，并在录音结束后注入到当前输入框。AtomVoice 不会保存你的识别文本历史记录。

3. **剪贴板内容**
   为了把识别文本输入到当前光标位置，AtomVoice 会临时使用系统剪贴板执行粘贴操作。应用会在注入前临时保存原剪贴板内容，注入后尽量恢复原内容。剪贴板内容仅在本机内存中短暂处理，不会上传到 AtomVoice 服务器。

4. **辅助功能相关信息**
   AtomVoice 使用 macOS 辅助功能权限监听触发键、检测当前输入位置，并模拟粘贴操作。应用不会记录你的按键内容，也不会持续读取其他应用中的文本内容。仅在必要时读取当前聚焦输入框的光标附近信息，用于避免重复标点等输入体验优化。

5. **本地设置**
   AtomVoice 会在本机保存应用设置，例如语言、识别引擎、触发键、输入设备、动画样式、静音停止设置、LLM 服务商地址、模型名称、自定义提示词等。这些设置保存在 macOS 本地偏好设置中。

6. **LLM API Key**
   如果你主动开启 LLM 文本优化并填写 API Key，AtomVoice 会将该 API Key 保存在本机设置中，并仅用于向你选择的 LLM 服务商发起请求。AtomVoice 不会把 API Key 上传到 AtomVoice 自有服务器。

## 3. 语音识别如何处理

AtomVoice 支持不同的识别方式：

1. **Apple 语音识别**
   默认情况下，AtomVoice 使用 Apple Speech 框架进行语音识别。根据 macOS、语言和系统能力，语音识别可能在设备端完成，也可能由 Apple 的语音识别服务处理。相关数据处理受 Apple 隐私政策约束。

2. **Apple 本机识别模式**
   如果你开启"Apple 本机识别"且当前语言支持，AtomVoice 会要求系统使用设备端识别。

3. **Sherpa ONNX 本地识别**
   如果你选择并配置本地 Sherpa ONNX 识别模型，音频识别会在本机完成，不需要上传到云端识别服务。

## 4. LLM 文本优化

LLM 文本优化默认关闭。

如果你主动开启该功能，AtomVoice 会将本次识别出的文本发送给你配置的 LLM 服务商，用于纠错、补全标点或优化语音转写错误。可能使用的服务商包括 OpenAI、Anthropic、DeepSeek、Moonshot、阿里云百炼、智谱 AI、零一万物、Groq、自定义 OpenAI-compatible API 或本地 Ollama。

发送给 LLM 服务商的数据通常包括：

1. 本次识别文本
2. 系统提示词或自定义提示词
3. 你配置的模型名称
4. 认证所需的 API Key

这些数据的处理方式由你选择的 LLM 服务商决定。请在使用前阅读对应服务商的隐私政策和数据使用条款。

如果你不启用 LLM 文本优化，AtomVoice 不会将识别文本发送给这些 LLM 服务商。

## 5. 自动更新检查

AtomVoice 会通过 GitHub Releases 检查是否有新版本。检查更新时，应用会向 GitHub 请求最新版本信息。GitHub 可能会根据其自身政策接收网络请求信息，例如 IP 地址、设备网络信息和 User-Agent。

AtomVoice 不会在更新检查中主动发送你的录音、识别文本、剪贴板内容或 LLM API Key。

## 6. 权限说明

AtomVoice 需要以下 macOS 权限：

1. **麦克风权限**
   用于录制你的语音并进行语音识别。

2. **语音识别权限**
   用于调用 Apple Speech 框架将语音转换为文本。

3. **辅助功能权限**
   用于监听触发键、检测输入位置，并将识别文本注入当前应用。

你可以随时在 macOS「系统设置」中撤销这些权限。撤销后，相关功能可能无法正常使用。

## 7. 数据保存与删除

AtomVoice 不保存录音文件，不保存语音识别历史记录，也不建立用户账户。

本地保存的数据主要是应用设置。你可以通过以下方式删除相关数据：

1. 在应用内清空或修改 LLM 设置
2. 删除 macOS 中 AtomVoice 的应用偏好设置
3. 删除应用及其相关本地支持文件

如果你使用第三方 LLM 服务或 Apple 语音识别服务，请根据对应服务商的政策管理或删除相关数据。

## 8. 数据共享

AtomVoice 不会出售、出租或交易你的个人数据。

只有在以下情况下，数据可能被发送至第三方：

1. 使用 Apple 语音识别时，音频或识别请求可能由 Apple 处理
2. 主动开启 LLM 文本优化时，识别文本会发送给你选择的 LLM 服务商
3. 检查更新时，应用会访问 GitHub Releases
4. 使用自定义 API 地址时，数据会发送至你自行配置的服务端

## 9. 安全措施

AtomVoice 尽量减少数据处理范围，并优先在本机完成任务。在线请求通常通过 HTTPS 发送，但如果你配置了自定义 API 地址，例如本地 Ollama 或其他 HTTP 地址，请自行确认该服务的安全性。

请妥善保管你的 LLM API Key，避免在不可信设备或共享账户环境中保存敏感凭据。

## 10. 儿童隐私

AtomVoice 面向一般 macOS 用户，不专门面向儿童。我们不会有意收集儿童的个人信息。

## 11. 隐私政策变更

我们可能会根据应用功能变化更新本隐私政策。重大变更会通过项目页面、发布说明或应用内说明进行提示。
