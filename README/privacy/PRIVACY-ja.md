# AtomVoice プライバシーポリシー

最終更新日：2026年5月5日

AtomVoice は macOS メニューバー音声入力ツールです。私たちはお客様のプライバシーを重視しています。本プライバシーポリシーでは、AtomVoice がデータをどのように処理し、権限を使用し、サードパーティサービスと連携するかを説明します。

## 1. 基本方針

AtomVoice は、デバイス上でローカルに処理を行い、データ収集を最小限に抑えるよう設計されています。

AtomVoice はユーザーアカウントを管理せず、広告を表示せず、分析 SDK を組み込まず、ユーザーの行動を追跡せず、個人情報を販売、貸与、共有することはありません。

## 2. 処理するデータ

AtomVoice は動作中に以下のデータを処理する場合があります：

1. **音声音声**
   トリガーキーを押して録音を開始すると、AtomVoice はマイクにアクセスし、音声認識と波形表示のために音声音声を処理します。録音終了後、AtomVoice は音声をローカルファイルに保存したり、AtomVoice のサーバーにアップロードしたりすることはありません。

2. **認識テキスト**
   音声認識結果は一時的にフローティングカプセルウィンドウに表示され、録音終了後に現在の入力フィールドに注入されます。AtomVoice は認識テキストの履歴を保存しません。

3. **クリップボードの内容**
   認識テキストを現在のカーソル位置に入力するために、AtomVoice はシステムクリップボードを使用して貼り付け操作を行います。アプリは注入前に元のクリップボードの内容を一時保存し、注入後に元の内容を復元します。クリップボードの内容はローカルメモリ上でのみ短時間保持され、AtomVoice のサーバーにはアップロードされません。

4. **アクセシビリティ関連情報**
   AtomVoice は macOS のアクセシビリティ権限を使用してトリガーキーを監視し、現在の入力位置を検出し、貼り付け操作をシミュレートします。アプリはキー入力を記録したり、他のアプリケーションのテキストを継続的に読み取ったりすることはありません。重複句読点の回避などの入力体験の最適化のために、現在の入力フィールドのカーソル付近の情報を必要な場合のみ読み取ります。

5. **ローカル設定**
   AtomVoice は言語、認識エンジン、トリガーキー、入力デバイス、アニメーションスタイル、サイレンス自動停止設定、LLM プロバイダーURL、モデル名、カスタムプロンプトなどのアプリ設定をローカルに保存します。これらの設定は macOS のローカル設定に保存されます。

6. **LLM API Key**
   LLM テキスト最適化を有効にして API Key を入力した場合、AtomVoice はその API Key をローカル設定に保存し、選択した LLM プロバイダーへのリクエストにのみ使用します。AtomVoice は API Key を AtomVoice のサーバーにアップロードすることはありません。

## 3. 音声認識の仕組み

AtomVoice は異なる認識モードをサポートしています：

1. **Apple 音声認識**
   デフォルトでは、AtomVoice は Apple Speech フレームワークを使用して音声認識を行います。macOS のバージョン、言語、システムの機能に応じて、音声認識はデバイス上で実行されるか、Apple の音声認識サービスによって処理されます。関連するデータの処理は Apple のプライバシーポリシーに従います。

2. **Apple デバイス上認識モード**
   「Apple デバイス上認識」を有効にし、現在の言語がサポートされている場合、AtomVoice はシステムにデバイス側での認識を要求します。

3. **Sherpa ONNX ローカル認識**
   ローカル Sherpa ONNX 認識モデルを設定した場合、音声認識はすべてデバイス上で実行され、クラウド認識サービスへのアップロードは不要です。

## 4. LLM テキスト最適化

LLM テキスト最適化はデフォルトで無効です。

この機能を有効にすると、AtomVoice は現在の認識テキストを設定した LLM プロバイダーに送信し、エラー修正、句読点の補完、音声転写の改善を行います。サポートされるプロバイダーには OpenAI、Anthropic、DeepSeek、Moonshot、Alibaba Cloud Bailian、Zhipu AI、Lingyi Wanwu、Groq、カスタム OpenAI-compatible API、ローカル Ollama が含まれます。

LLM プロバイダーに送信されるデータには通常以下が含まれます：

1. 現在のセッションの認識テキスト
2. システムプロンプトまたはカスタムプロンプト
3. 設定したモデル名
4. 認証用 API Key

このデータの処理方法は、選択した LLM プロバイダーによって異なります。使用前に各プロバイダーのプライバシーポリシーとデータ利用規約をご確認ください。

LLM テキスト最適化を有効にしない場合、AtomVoice は認識テキストを LLM プロバイダーに送信しません。

## 5. 自動更新チェック

AtomVoice は GitHub Releases を通じて新しいバージョンを確認します。更新チェック時、アプリは最新バージョン情報を GitHub に要求します。GitHub は独自のポリシーに従い、IP アドレス、デバイスのネットワーク情報、User-Agent などのネットワークリクエスト情報を受信する場合があります。

AtomVoice は更新チェック時に録音データ、認識テキスト、クリップボードの内容、LLM API Key を送信することはありません。

## 6. 必要な権限

AtomVoice は以下の macOS 権限を必要とします：

1. **マイク権限**
   音声認識のために音声を録音するために使用します。

2. **音声認識権限**
   Apple Speech フレームワークを呼び出して音声をテキストに変換するために使用します。

3. **アクセシビリティ権限**
   トリガーキーの監視、入力位置の検出、認識テキストの現在のアプリケーションへの注入に使用します。

macOS のシステム設定からいつでもこれらの権限を取り消すことができます。権限を取り消すと、関連機能が正常に動作しなくなる場合があります。

## 7. データの保存と削除

AtomVoice は音声録音を保存せず、音声認識履歴を保存せず、ユーザーアカウントを作成しません。

ローカルに保存されるデータは主にアプリ設定です。以下の方法で関連データを削除できます：

1. アプリ内で LLM 設定をクリアまたは変更する
2. macOS で AtomVoice のアプリ設定を削除する
3. アプリとその関連ローカルサポートファイルを削除する

サードパーティの LLM サービスや Apple の音声認識を使用している場合は、各プロバイダーのポリシーに従って関連データを管理または削除してください。

## 8. データの共有

AtomVoice は個人データを販売、貸与、取引することはありません。

データがサードパーティに送信されるのは以下の状況のみです：

1. Apple 音声認識を使用する場合、音声または認識リクエストは Apple によって処理される場合があります
2. LLM テキスト最適化が有効な場合、認識テキストが選択した LLM プロバイダーに送信されます
3. 更新チェック時、アプリは GitHub Releases にアクセスします
4. カスタム API エンドポイントを使用する場合、データは設定したサーバーに送信されます

## 9. セキュリティ対策

AtomVoice はデータ処理を最小限に抑え、デバイス上での処理を優先します。オンラインリクエストは通常 HTTPS で送信されますが、ローカル Ollama やその他の HTTP アドレスなどのカスタム API エンドポイントを設定した場合、そのサービスのセキュリティを確認してください。

LLM API Key を適切に管理し、信頼できないデバイスや共有アカウント環境に認証情報を保存しないでください。

## 10. 子供のプライバシー

AtomVoice は一般の macOS ユーザーを対象としており、子供を特に対象としたものではありません。子供の個人情報を意図的に収集することはありません。

## 11. ポリシーの変更

アプリの機能変更に応じて、本プライバシーポリシーを更新する場合があります。重要な変更はプロジェクトページ、リリースノート、またはアプリ内のお知らせを通じて通知します。
